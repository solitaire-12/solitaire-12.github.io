从数据库中对数据表名称进行整理，结果如下[*指通配符]：

  16. FLW_RU_*

从上述整理结果看，表名用_可分为前缀，中间，后缀三个部分。

前缀

不难看出，前缀分为两种：

  • ACT_
  • FLW_

ACT_，由于Flowable 基于 Activiti 流程引擎开发而来，延用Activiti 数据表的部分，即ACT_指 Activiti；FLW_是Flowable流程引擎特有的数据表。

中间

名称 含义
APP 应用程序相关的表
CMMN CMMN协议相关的表
CO（CONTENT） 内容引擎相关的表
DMN DMN协议相关的表
EVT（EVENT） 事件相关的表
FO（FORM） 表单相关的表
GE（GENERAL） 通用表，用于各种用例
HI（HISTORY） 历史数据相关的表，比如当运行时表中删除数据时，历史表仍然包含这些已完成实例的所有信息
ID（IDENTITY） 用户身份认证相关的表
PROCDEF
（PROCESSDEFINE）		 记录流程定义相关的表
RE（REPOSITORY） 流程定义、流程资源等包含静态信息相关的表
RU（RUNTIME） 运行时相关的表，比如未完成的流程、案例等的运行时数据的运行表。
Flowable仅在执行期间存储运行时数据，并在实例结束后删除记录，以此保证运行时表保持小而快。
CHANNEL 泳道相关的表
EV
EVENT 事件相关的表

后缀

DATABASECHANGELOG：表示该表是Liquibase执行的记录，Liquibase是一个数据库脚本管理工具。

DATABASECHANGELOGLOCK：表示该表记录Liquibase执行锁的，用来确保一次只运行一个Liquibase实例。

后续有这两个的，用 🇩🇪 代替

DEPLOYMENT：[中间部分_ ]部署时，会存储一条记录以保存此部署。部署的实际内容被存储在[中间部分_DEPLOYMENT_RESOURCE]表中，并在该表中引用。

DEPLOYMENT_RESOURCE：包含构成应用程序部署的实际资源（资源为字节）。当引擎需要实际模型时，将从该表中获取资源。

后续有这两个后缀的，用 🇩🇲 代替

  • ACT_APP_*：负责应用引擎存储和应用部署定义
    • ACT_APP_APPDEF：应用程序模型产生应用程序定义，比如流程/案例等。是成功部署到应用引擎的应用模型的表示。
    • 🇩🇲
    • ACT_APP_PAGEDEF
    • 🇩🇪
  • ACT_CMMN_*：Flowable CMMN Egine的数据库以ACT_CMMN_*开头，和BPMN一样，是一种流程内容的规范。一般用于存储处理BPMN不适用的业务场景数据。只要符合CMMN规范的模型数据才会使用该表。
    • ACT_CMMN_CASEDEF
    • 🇩🇲
    • ACT_CMMN_HI_CASE_INST：CMMN引擎启动的每个案例实例数据
    • ACT_CMMN_HI_MIL_INST：在案例实例中达到的每个里程碑数据
    • ACT_CMMN_HI_PLAN_ITEM_INST：案例实例执行的一部分创建的每个计划实例的数据。
    • ACT_CMMN_RU_CASE_INST：每个已启动但尚未完成的案例实例的条目。
    • ACT_CMMN_RU_MIL_INT：运行实例的一部分达到的每个里程碑的条目
    • ACT_CMMN_RU_PLAN_ITEM_INST：在案件实例执行期间创建的每个实例的条目
    • ACT_CMMN_RU_SENTRY_PART_INST：计划项目实例有可能有守卫状态转换的哨兵，该表用来存储哨兵数据。
    • 🇩🇪
  • ACT_CO_*：
    • ACT_CO_CONTENT_ITEM：每项内容在此表中都有个条目
    • 🇩🇪
  • ACT_DMN_*：Flowable DMN的数据库名称以此开头。
    • 🇩🇲
    • ACT_DMN_DECISION：存储已部署决策表的元数据，并与来自其他引擎的定义相对应。
    • ACT_DMN_HI_DECISION_EXECUTION：存储有关DMN决策表执行的审计信息。
  • ACT_EVT_*：
    • ACT_EVT_LOG：Flowable引入了事件日志机制，默认会在数据库创建该表，用来存储事件日志，如果不适用事件日志，可以删除该表。
  • ACT_FO_FORM_*：存储表单引擎和围绕表单模型和这些表单的实例数据。
    • 🇩🇪
    • ACT_FO_FORM_DEFINITION：表单定义表
    • ACT_FO_FORM_DEPLOYMENT：表单部署表
    • ACT_FO_FORM_INSTANCE：表单实例表
    • ACT_FO_FORM_RESOURCE：表单元数据表
  • ⭐ACT_GE_*：
    • ACT_GE_BYTEARRAY：存储每个流程的部署记录，bytes_ 字段中保存流程的具体内容。
    • ACT_GE_PROPERTY：存储Flowable自身变量，如版本号。
  • ⭐ACT_HI_*：存储正在运行和已完成的实例的历史数据。
    • ACT_HI_ACTINST：历史活动信息表。这里记录流程流转的所有节点，与只记录task内容的ACT_HI_TASKINST有所不同。
    • ACT_HI_ATTACHMENT：历史附件表
    • ACT_HI_COMMENT：流程的历史评论表
    • ACT_HI_DETAIL：历史详情表，流程中产生的变量详细，包括流程流转的变量，业务表单中填写的流程需要用到的变量等。
    • ACT_HI_ENTITYLINK：历史参与的人员表
    • ACT_HI_IDENTITYLINK：任务参与者数据表，主要存储历史节点参与者的信息，可能时Group也可能是User
    • ACT_HI_PROCINST：保存每一个历史流程，创建时就生成，一条流程实例对应一条记录。
    • ACT_HI_TASKINST：记录每个历史节点，一个task对应一条记录
    • ACT_HI_TSK_LOG：存储task每一次执行可能会带上的数据
    • ACT_HI_VARINST：流程历史变量表
  • ACT_ID_*：
    • ACT_ID_BYTEARRAY：用户组的部署内容
    • ACT_ID_GROUP：用户组表
    • ACT_ID_INFO：用户的信息，账号密码等
    • ACT_ID_MEMBERSHIP：用户和用户组的关联表
    • ACT_ID_PRIV：权限表
    • ACT_ID_PRIV_MAPPING：用户、用户组以及权限之间的关系表
    • ACT_ID_PROPERTY：用户的变量表
    • ACT_ID_TOKEN：用户的访问记录表
    • ACT_ID_USER：用户表
  • ACT_PROCDEF_*
    • ACT_PROCDEF_INFO：流程定义信息，对流程的说明
  • ⭐ACT_RE_*：和流程定义、流程资源等包含静态信息相关内容
    • ACT_RE_DEPLOYMENT：流程部署记录。每次服务重启部署一次，会在这里新增一条记录
    • ACT_RE_MODEL：创建模型时，额外定义的一些模型相关信息，存在这张表，默认不保存。
    • ACT_RE_PROCDEF：记录流程的变更，流程每变更一次存一条记录，version_ 字段加1。
  • ⭐ACT_RU_*：与流程引擎运行时信息相关
    • ACT_RU_ACTINST：流程中每个活动在此表中都有一条记录来指示活动的当前状态。
    • ACT_RU_JOB
      ACT_RU_TIMER_JOB
      ACT_RU_SUSPENDED_JOB
      ACT_RU_EXTERNAL_JOB
      ACT_RU_HISTORY_JOB
      ACT_RU_DEADLETTER_JOB
      Flowable引擎使用作业表来实现异步逻辑、计时器或历史处理。这些表存储每个作业所需数据。
    • ACT_RU_ENTITYLINK：存储有关实例的父子关系的信息。如：流程实例启动子案例实例，则此关系存储在表中。
    • ACT_RU_EVENT_SUBSCR：当流程定义使用事件（信号/消息/等或启动/中间/边界）时，引擎将对该表的引用存储在该表中。
    • ACT_RU_EXECUTION：存储流程实例和指向流程实例当前状态的指针（称作执行）。
    • ACT_RU_IDENTITYLINK：存储有关用户或组的数据及其与（流程/案例/等）实例相关的角色
    • ACT_RU_TASK：保存正在运行的实例和每个未完成用户任务的条目，在查询用户的任务列表时使用此表。CMMN也用
    • ACT_RU_VARIABLE：保存与实例相关的变量。CMMN也用
  • FLW_CHANNEL_*
    • FLW_CHANNEL_DEFINITION
  • FLW_EV_*

04.Flowable业务层

Flowable Engine提供流程相关的一些接口，从flowable-engine的 jar 包中有个ProcessEngine接口，其源码摘录如下：

ProcessEngine 是 Flowable 引擎的核心接口，它是流程引擎的主入口，同时负责与各个服务进行交互和管理流程的整个生命周期。下面简单介绍ProcessEngine接口内容：

  • String VERSION = "6.8.0.0"，交代Flowable的版本。
  • void startExecutors()：启动所有流程引擎的执行器，执行器用于处理流程实例的执行，在引擎启动时，执行器会自动运行并处理代办任务和定时任务。
  • RepositoryService getRepositoryService()：获取流程仓库服务对象（RepositoryService），用于管理流程定义的部署和查询。
  • RuntimeService getRuntimeService()：获取运行时服务对象（RuntimeService），用于操作流程的运行是数据，如启动流程实例、执行流程任务、查询流程实例状态等。
  • FormService getFormService()：获取表单服务对象（FormService），用于管理表单数据和处理与表单相关的操作，如获取表单内容、保存表单数据等。
  • TaskService getTaskService()：获取任务服务对象（TaskService），用于管理任务数据和处理与任务相关的操作，如创建任务、完成任务、查询任务列表等。
  • HistoryService getHistoryService()：获取历史数据服务对象（HistoryService），用于查询流程的历史数据，如查询已完成的流程实例、查询历史任务等。
  • IdentityService getIdentityService()：获取身份验证和授权服务对象（IdentityService)，用于管理用户、组、角色等身份信息，以及进行权限和身份验证。
  • ManagementService getManagementService()：获取管理服务对象（ManagementService)，用于进行底层的引擎管理和操作，如数据库管理、作业管理、引擎配置等。
  • DynamicBpmnService getDynamicBpmnService()：获取动态BPMN服务对象（DynamicBpmnService)，用于动态修改流程定义的流程元素，如添加活动、删除活动、修改连线等。
  • ProcessMigrationService getProcessMigrationService()：获取流程迁移服务对象，用于支持流程定义的迁移操作，如迁移流程实例、修改流程定义版本等。
  • ProcessEngineConfiguration getProcessEngineConfiguration()：获取流程引擎的配置对象（ProcessEngineConfiguration)，可以通过配置对象进行流程引擎的详细配置和定制，如数据库配置、作业调度配置、缓存配置等。
